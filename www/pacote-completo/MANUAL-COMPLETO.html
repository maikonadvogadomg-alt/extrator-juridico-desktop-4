1. Visão Geral do Projeto

O Extrator Jurídico é uma ferramenta standalone — um único arquivo HTML que roda diretamente no navegador, sem servidor, sem instalação, sem dependências externas próprias.

🔍 Extrator de Sites

Acessa qualquer site jurídico via proxies públicos e extrai scripts de cálculo (INSS, previdenciário, trabalhista). Baixa os scripts como .js, HTML completo ou relatório .txt.

🤖 IA Integrada

Assistente "Jasmim" analisa os scripts extraídos. Aceita qualquer provedor de IA via chave direta. Auto-detecção pelo prefixo da chave.

📦 APK Android

O arquivo index.html pode ser embalado como APK via Capacitor + Android Studio, ou compilado na nuvem via GitHub Actions sem instalar nada.

🖥 Executável PC

Pode ser empacotado como aplicativo Desktop (.exe para Windows, .dmg para Mac, .AppImage para Linux) via Electron.

3. Instalação — Todos os Ambientes

A. Rodar direto no navegador — sem instalar NADA

Esta é a forma mais simples. Funciona em qualquer computador ou celular com Chrome/Firefox/Edge.
  1. Extraia o arquivo extrator-juridico-completo.zip em qualquer pasta
  2. Windows: clique duas vezes em iniciar.bat
  3. Mac / Linux: abra o terminal na pasta e execute ./iniciar.sh
  4. Qualquer sistema: arraste o arquivo index.html para o Chrome
  5. O app abre. Pronto.
Nota CORS no Chrome: Para funcionar corretamente em arquivo local, use Chrome ou Firefox. O Safari pode bloquear algumas requisições de arquivo local. Se tiver problemas, use: chrome --disable-web-security --user-data-dir=/tmp/chrome (apenas para desenvolvimento)

Passo a passo

  1. Extraia o ZIP do pacote completo
  2. Abra o terminal na pasta extraída e instale as dependências: 
    npm install
  3. Copie os arquivos web para o projeto Android: 
    npx cap copy android
  4. Abra o Android Studio → File → Open → selecione a pasta android/
  5. Aguarde o Gradle sincronizar (~5 minutos na primeira vez). O Android Studio vai baixar dependências automaticamente.

  7. Quando terminar, clique em "Locate" ou encontre o APK em: 
    android/app/build/outputs/apk/debug/app-debug.apk
  8. Transfira o .apk para o celular e instale

C. Gerar APK — via GitHub Actions (sem instalar NADA)

Esta é a forma mais fácil. Você só precisa de uma conta no GitHub. O APK é compilado na nuvem em ~10 minutos.
  1. Crie uma conta gratuita em github.com (se não tiver)
  2. Crie um novo repositório: New → Repository name → Create repository
  3. Extraia o ZIP do pacote completo no seu computador
  4. Instale o Git (git-scm.com) e execute na pasta extraída: 
    git init
git add .
git commit -m "Extrator Jurídico v2.0"
git branch -M main
git remote add origin https://github.com/SEU-USUARIO/SEU-REPOSITORIO.git
git push -u origin main
  5. No GitHub, clique na aba Actions → você verá o workflow rodando automaticamente
  6. Aguarde ~10-15 minutos até terminar (ícone verde ✓)
  7. Clique em Releases (na barra lateral direita do repositório) → latest-apk → baixe o arquivo .apk
  8. Instale no celular: transfira o .apk → Configurações → Segurança → Fontes desconhecidas ✓ → abra o .apk
Atualizações automáticas: Toda vez que fizer git push com alterações, o GitHub Actions compila um novo APK automaticamente. Sem precisar instalar nada.

4. Rotas de API Utilizadas

Proxies Públicos — Site Extrator

Todos os proxies são gratuitos, públicos e sem necessidade de chave ou conta.
#ProxyURL baseFormato de chamadaResposta
1 (primário) AllOrigins api.allorigins.win GET /get?url={URL_CODIFICADA} JSON: { "contents": "html…" }
2 (fallback) CORSProxy.io corsproxy.io GET /?{URL_CODIFICADA} HTML direto
3 (fallback) CodeTabs api.codetabs.com GET /v1/proxy?quest={URL_CODIFICADA} HTML direto

O sistema tenta automaticamente o Proxy 1. Se falhar (timeout 14s ou erro HTTP), tenta o Proxy 2, depois o Proxy 3. Se todos falharem, exibe erro com sugestão.

Exemplo real de chamada (AllOrigins)

GET https://api.allorigins.win/get?url=https%3A%2F%2Fwww.calculojuridico.com.br

Resposta:
{
  "contents": "<!DOCTYPE html><html>...</html>",
  "status": { "url": "https://www.calculojuridico.com.br", "content_type": "text/html", "http_code": 200 }
}

Limitações dos proxies públicos

  • Sites com Cloudflare Enterprise ou Bot Fight Mode podem bloquear — isso é limitação do site, não do extrator
  • Rate limit: ~60 req/min por IP (AllOrigins). Para uso normal, mais que suficiente.
  • Sites com conteúdo carregado por JavaScript (SPA sem SSR) podem retornar HTML vazio — os scripts estarão nos arquivos .js externos

8. GitHub Actions — Workflow Completo

O arquivo .github/workflows/build-apk.yml compila o APK automaticamente sempre que você faz push no branch main ou master.

O que o workflow faz (passo a passo)

  1. Faz checkout do código (actions/checkout@v4)
  2. Instala Java 17 com Temurin distribution
  3. Configura Gradle 8.6
  4. Instala o Android SDK com android-actions/setup-android@v3
  5. Instala plataforma Android 34 e build-tools 34.0.0 via sdkmanager
  6. Instala Node.js 20
  7. Instala @capacitor/cli, @capacitor/core e @capacitor/android versão 6.1.2
  8. Executa npx cap copy android para copiar o HTML para o projeto Android
  9. Executa ./gradlew assembleDebug --no-daemon (timeout: 35 min)
  10. Localiza o .apk gerado
  11. Deleta a release anterior (latest-apk)
  12. Publica nova release com o APK como attachment

Variáveis de ambiente usadas pelo workflow

VariávelValorDescrição
GRADLE_OPTS-Xmx3g -XX:MaxMetaspaceSize=512mHeap do Gradle (3 GB — necessário para compilar Capacitor)
JAVA_TOOL_OPTIONS-Xmx3gHeap do Java compiler
GH_TOKEN${{ secrets.GITHUB_TOKEN }}Token automático do GitHub Actions (sem configurar nada)
O GITHUB_TOKEN é gerado automaticamente pelo GitHub Actions a cada execução. Você não precisa configurar nenhum secret adicional para compilar o APK.

9. Configuração da IA — Todos os Provedores

  1. No app, clique no botão 🤖 (canto inferior direito) para abrir o chat da IA
  2. Clique em 🔑 (dentro do painel da IA) para abrir a configuração da chave
  3. Cole a chave no campo "Chave API". O provedor é detectado automaticamente pelo prefixo
  4. Verifique o texto em verde que confirma qual provedor foi detectado
  5. Opcionalmente: informe o modelo (deixe em branco para usar o padrão)
  6. Clique em ✅ Salvar. A chave fica salva no localStorage do navegador

Como testar se a IA está funcionando

  1. No chat, escreva: Olá, responda em português
  2. Clique em Enviar ou pressione Ctrl+Enter
  3. Se retornar uma resposta, está funcionando
  4. Se retornar erro 401, a chave está errada ou expirada
  5. Se retornar erro 429, excedeu o limite de requisições grátis — aguarde ou troque de provedor

Usando múltiplos provedores

A chave é salva no localStorage com a chave extrator-ai-key. Para trocar de provedor, clique em 🔑 e cole a nova chave. A detecção automática reconhece o novo provedor e ajusta a URL e modelo.

11. Solução de Problemas

O site não carregou / "Todos os proxies falharam"

  • O site usa Cloudflare com Bot Fight Mode → não é possível contornar via proxy público. Solução: acesse o site normalmente no navegador, salve a página com Ctrl+S e analise o HTML salvo.
  • URL errada → certifique que a URL existe e está acessível no navegador
  • Tente sem www (ex: calculojuridico.com.br em vez de www.calculojuridico.com.br)
  • Tente com profundidade 1 (menos requisições)

Nenhum script jurídico foi encontrado

  • O site pode ser uma SPA (Single Page App) que carrega o conteúdo via JavaScript — o HTML retornado pelo proxy estará vazio. Os scripts estarão nos arquivos .js externos (aba Assets → Scripts JS → clique 👁 para ver)
  • Tente profundidade 2 para buscar subpáginas

Erro de IA: 401 Unauthorized

  • Chave de API inválida ou expirada
  • Gere uma nova chave no painel do provedor

Erro de IA: 429 Too Many Requests

  • Excedeu o limite grátis (Groq: 30 req/min · Gemini: 15 req/min)
  • Aguarde 1 minuto ou troque para o outro provedor grátis

O GitHub Actions falhou

  • Clique no workflow falhado → expanda o passo com ícone ✕ vermelho → leia o erro
  • Erros comuns: SDK not found → o passo de SDK falhou (tente novamente via "Re-run all jobs")
  • Gradle sync failed → verifique se os arquivos build.gradle estão no repositório

APK instalou mas a tela está branca

  • O index.html não está na pasta dist/ do projeto
  • Certifique que executou npx cap copy android antes de compilar
  • No workflow do GitHub Actions, isso é feito automaticamente

A chave da IA sumiu / preciso reconfigurar

  • A chave é salva no localStorage do navegador. Se limpou o cache ou abriu em outro navegador, precisa configurar de novo
  • No APK Android, o localStorage persiste entre sessões normalmente
